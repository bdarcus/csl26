Advanced Dates (EDTF)

CSLN uses Extended Date/Time Format (EDTF) for native support of uncertainty, approximation, and ranges. Configure style-specific rendering preferences declaratively.

Locale Configuration (en-US.yaml) 

dates:
  uncertainty-term: uncertain
  open-ended-term: present

terms:
  circa:
    long: circa
    short: c.

Style Configuration (Optional Overrides) 

options:
  dates:
    uncertainty-marker: "?"           # or null to suppress
    approximation-marker: "ca. "       # uses locale circa term
    range-delimiter: "–"              # en-dash (U+2013)
    open-range-marker: null           # uses locale open-ended-term

bibliography:
  template:
    - date: issued
      form: full                      # year | year-month | full
      wrap: parentheses

EDTF Strings & Rendered Output

Uncertain Date
EDTF: "1990?"
1990?
Displays uncertainty marker (Chicago) or suppresses (APA)
Approximate Date
EDTF: "1990~"
ca. 1990
Uses locale circa term (short form) or style override
Date Range
EDTF: "1990/2000"
1990–2000
Range delimiter defaults to en-dash
Mixed Uncertainty Range
EDTF: "1990~/2000?"
ca. 1990–2000?
Combines approximation and uncertainty markers
Open-Ended Range
EDTF: "1990/.."
1990–present
Uses locale open-ended-term ("present" in en-US)
Year-Month Precision
EDTF: "1990-05/2000-12"
May 1990–December 2000
Preserves precision for range endpoints
info
EDTF Standard: CSLN implements Level 1 EDTF (ISO 8601-2) via the edtf Rust crate. Uncertainty (?), approximation (~), and range (/) metadata is preserved and rendered according to locale terms and style overrides.

Locale vs. Style: Locales provide language-specific default terms (circa, open-ended-term). Styles can override these with custom markers or suppress them entirely (e.g., APA suppresses uncertainty markers).

Multilingual Support

CSLN handles multilingual metadata with BCP 47 language tags, supporting original scripts, transliterations (Pinyin, Hepburn, ALA-LC), and translations. Styles declaratively specify rendering preferences with automatic fallback.

Style Configuration 

options:
  multilingual:
    title-mode: transliterated  # primary | transliterated | translated | combined
    name-mode: transliterated
    preferred-script: Latn       # Latin script for romanization

Reference Data (YAML) 

id: war-peace-zh
type: book
title:
  original: "战争与和平"
  lang: "zh"
  transliterations:
    zh-Latn-pinyin: "Zhànzhēng yǔ Hépíng"
  translations:
    en: "War and Peace"
author:
  original:
    family: "Толстой"
    given: "Лев"
  lang: "ru"
  transliterations:
    Latn:
      family: "Tolstoy"
      given: "Leo"

Rendered Output by Mode

Primary Mode
战争与和平. Лев Толстой (1869)
Original scripts preserved
Transliterated Mode
Zhànzhēng yǔ Hépíng. Leo Tolstoy (1869)
Romanized for Western readers
Translated Mode
War and Peace. Leo Tolstoy (1869)
English translation when available
Combined Mode
Zhànzhēng yǔ Hépíng [War and Peace]. Leo Tolstoy (1869)
Both romanization and translation
info
BCP 47 Fallback Strategy:
1. Exact tag match ("ja-Latn-hepburn")
2. Substring match ("Latn" matches any Latin script)
3. Fallback to original field

Titles & Links

CSLN separates title content from its role (primary, parent-monograph, parent-serial) and uses declarative global formatting rules for consistency across item types.

Global Formatting (options.titles) 

titles:
  monograph:
    emph: true      # Italics for books
  periodical:
    emph: true      # Italics for journals
  component:
    emph: false     # Plain for articles

Template Usage & Links 

template:
  - title: primary
    links:
      target: doi
      anchor: component
  - title: parent-serial
    suffix: ","
settings_input_component

Type-Specific Overrides

Avoid complex conditional logic. Components in CSLN can declaratively override their behavior or suppress themselves based on the item type (e.g., article, book, webpage). 

bibliography:
  template:
    - variable: publisher
      overrides:
        article-journal: { suppress: true }
        webpage:         { suppress: true }
    - number: pages
      overrides:
        chapter:
          prefix: "pp. "
          wrap: parentheses

Context & Presets

CSLN supports "presets" (reusable style foundations) and distinct templates for narrative (integral) vs. parenthetical (non-integral) citations. 

citation:
  use-preset: apa
  integral:      # Narrative: Smith (2024)
    template:
      - contributor: author
      - date: issued
        wrap: parentheses
        prefix: " "
  non-integral:  # Parenthetical: (Smith, 2024)
    wrap: parentheses
    template:
      - contributor: author
      - date: issued
      delimiter: comma

Options-Level Presets

Common configuration patterns for contributors, dates, and titles can be expressed as a single preset name instead of spelling out every field. Mix preset shorthand with explicit overrides for maximum clarity and concision. 

# Named presets replace verbose config blocks
options:
  contributors:            # Explicit config (verbose)
    display-as-sort: all
    initialize-with: ". "
    delimiter: ", "
    and: symbol
  dates: long              # Preset shorthand
  titles: apa             # Preset shorthand

# Available presets:
#   contributors: apa | chicago | harvard | springer | vancouver
#   dates:        long | short | numeric | iso
#   titles:       apa | chicago | humanities | ieee | scientific

Binary Performance (CBOR)

For production environments, CSLN styles and bibliographies can be compiled into binary CBOR (Concise Binary Object Representation) for near-instant loading.

Compilation

$ csln convert apa.yaml --output apa.cbor

Reduces parse time by up to 90% for large styles.

Processing

$ csln process data.cbor style.cbor

Full binary pipeline for maximum throughput.

Pluggable Output Formats

CSLN supports multiple output formats with a pluggable architecture. Generate semantic HTML, flexible Djot for static site generators, or clean Plain Text.

Plain Text (--format plain)
Smith, J. A. (2023). The Future of Citations. Academic Press.
Semantic HTML (--format html) 
<span class="csln-entry">
  <span class="csln-author">Smith, J. A.</span> (2023).
  <i class="csln-title">The Future of Citations</i>. Academic Press.
</span>
Djot (--format djot) 
[Smith, J. A.]{.csln-author} (2023). _The Future of Citations_{.csln-title}. Academic Press.
rocket_launch
Future Roadmap: The pluggable renderer system is designed to easily accommodate new targets like LaTeX and Typst, providing consistent citation rendering across different publishing toolchains.

Document Processing (Djot)

CSLN can process full documents by extracting citations using a high-performance winnow parser. It supports the proposed Djot citation syntax, including multi-cites and narrative modes.

Djot Input (paper.djot) 
According to @kuhn1962[p. 10]{.cite}, revolutions are...
Multi-cite: [see ;@kuhn1962; @watson1953, ch. 2]{.cite}.
Rendered Output (Plain Text)
According to Kuhn (1962, p. 10), revolutions are...
Multi-cite: (see Kuhn, 1962; Watson & Crick, 1953, ch. 2).
